# 产品需求文档（PRD）

## 项目背景

Windows 用户管理多个 WSL 实例时，需要频繁记忆和输入 `wsl.exe` 命令。本项目旨在提供一个简洁、直观的图形化管理界面，让启动、停止、安装、重命名等常用操作可以通过点击完成。

## 目标用户

- 在 Windows 上同时使用多个 Linux 发行版的开发者
- 需要快速查看 WSL 状态、释放资源或切换实例的用户
- 不熟悉 `wsl.exe` 命令行参数的新手用户

## 用户故事

- 作为开发者，我想打开应用就看到所有 WSL 实例的状态，以便快速决定启动哪个。
- 作为用户，我想点击按钮启动/停止实例，而不用打开 PowerShell。
- 作为用户，我想删除不用的实例前看到确认提示，避免误删数据。
- 作为开发者，我想一键打开 Windows Terminal 进入指定实例。
- 作为用户，我想在图形界面中直接安装新的 WSL 发行版并自定义实例名。
- 作为用户，我想重命名已有实例而不丢失数据。

## 功能需求

### P0 - MVP（必须有）

MVP 版本聚焦最核心的 WSL 管理操作，保证命令封装正确、参数安全、UI 可用。

| 编号 | 功能 | 说明 | 实际命令 |
| --- | --- | --- | --- |
| F01 | 实例列表 | 显示所有 WSL 实例的名称、状态、WSL 版本、默认标记、发行版名称、WSL2 VM IP | `wsl --list --verbose` + `wsl -d <name> -e ...` |
| F02 | 启动实例 | 启动已停止的实例（不打开可见窗口） | `wsl -d <name> -e true` |
| F03 | 停止实例 | 停止正在运行的实例 | `wsl --terminate <name>` |
| F04 | 停止全部 | 停止所有运行中的实例及 WSL 2 VM | `wsl --shutdown` |
| F05 | 删除实例 | 注销实例，需二次确认 | `wsl --unregister <name>` |
| F06 | 设置默认实例 | 设置默认 WSL 发行版 | `wsl --set-default <name>` |
| F07 | 打开终端 | 调用 Windows Terminal 进入实例 Shell | `wt.exe wsl -d <name>` |
| F08 | WSL 版本信息 | 显示 WSL 本身版本、内核版本、Windows 版本，支持中英文输出 | `wsl --version` / `wsl --status` |
| F09 | 创建实例 | 从 Microsoft Store / 在线源安装发行版，支持自定义实例名 | `wsl --install -d <distro> --name <name> --no-launch` |
| F10 | 重命名实例 | 通过 export + import + unregister 重命名，保留数据 | `wsl --export` + `wsl --import` + `wsl --unregister` |
| F11 | 刷新状态 | 手动刷新列表，异步不阻塞 UI | `wsl --list --verbose` |

### P1 - 重要（MVP 不做）

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F12 | 导入/导出实例 | `wsl --import` / `--export`，支持选择路径与 `--vhd` |
| F13 | 切换 WSL 版本 | `wsl --set-version <name> 1/2`，耗时且常需管理员 |
| F14 | 磁盘占用显示 | 估算实例虚拟磁盘大小 |
| F15 | 更多详情 | 运行时间、资源占用等 |
| F16 | 搜索/过滤 | 按名称、状态、发行版过滤 |

### P2 - 可选（后续版本）

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| F17 | 全局热键 | 快速调出管理器 |
| F18 | 系统托盘 | 最小化到托盘、状态指示灯 |
| F19 | 快照/备份调度 | 定时导出实例 |
| F20 | 多语言界面 | 中文/英文界面切换 |

## 非目标

- 不替代 Windows Terminal 或 WSL 本身。
- 不做跨平台（仅限 Windows 10/11）。
- 不管理 Docker Desktop 内置的 WSL 实例（后续可默认隐藏）。
- MVP 不做导入/导出、版本切换。

## 性能需求

- 应用冷启动 <= 3 秒
- 实例列表刷新 <= 2 秒
- UI 操作反馈 <= 200ms
- 所有耗时操作（刷新、创建、启动/停止、重命名等）必须异步执行，不阻塞主界面交互

## 兼容性

- Windows 10 版本 1903 及以上（WSL 2 需要 2004 及以上）
- WSL 版本 1 与 2
- 支持 x64 与 ARM64 Windows
