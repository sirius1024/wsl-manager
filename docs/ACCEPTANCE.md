# 验收标准

## 通用验收规则

- 所有 P0 功能必须通过人工或自动化测试。
- 无已知阻塞性 bug。
- 前端 TypeScript 类型检查通过（`npm run build`）。
- Rust 单元测试通过（`cargo test`）。
- 危险操作均有二次确认。
- 在至少一台 Windows 物理机/虚拟机中实测通过。

## P0 功能验收清单

| 功能 | 验收标准 |
| --- | --- |
| 实例列表 | 启动应用后 2 秒内展示所有 WSL 实例，状态正确 |
| 启动实例 | 点击启动后，状态从 Stopped 变为 Running |
| 停止实例 | 点击停止后，状态从 Running 变为 Stopped |
| 停止全部 | 二次确认后所有 Running 实例停止，WSL 2 VM 关闭 |
| 删除实例 | 二次确认后删除成功，列表刷新，不删除其它实例 |
| 设置默认 | 默认实例标记更新，重启应用后保持 |
| 打开终端 | 调用 Windows Terminal 并进入实例 Shell |
| 安装发行版 | 选择发行版、输入实例名后成功触发安装，列表可刷新出新实例 |
| 重命名实例 | 二次确认后实例名变更，数据保留，默认标记保持 |
| WSL 版本信息 | 正确显示 WSL / 内核 / Windows 版本 |

## 测试类型

1. **Rust 单元测试**：WSL 命令封装、参数校验、输出解析（`cargo test`）。
2. **前端类型/构建检查**：`npm run build`。
3. **集成测试**：在已安装 WSL 的 Windows 环境中执行真实命令。
4. **手动测试**：按下方清单执行。

## 手动测试场景

- [ ] 安装 WSL 但未安装任何发行版时打开应用
- [ ] 存在 Running 和 Stopped 混合实例时操作
- [ ] 删除实例时点击取消，确认数据未被删除
- [ ] 重命名实例时点击取消，确认原实例未被修改
- [ ] 网络断开或 WSL 服务异常时的错误提示
- [ ] 非管理员用户执行普通操作无需提权
- [ ] 非 Windows 系统开启 mock 模式后 UI 正常显示

## 性能验收

- 冷启动 <= 3 秒
- 列表刷新 <= 2 秒
- 普通操作 UI 反馈 <= 200ms
